Text Conferencing Server

  • by: Justin Bauer
  • date: Fall 2021
  • technologies: C, TCP

In my Computer Networks course (ECE361) at UofT, my partner and I developed a text conferencing application for the linux command line. There are two main components to our application, the server and the client.

The server listens for new clients on a chosen port. Clients can connect to the server by executing a login command with the server address and port number as well as login credentials. The server validates these credentials and ensures the clients are known. Once the clients are connected, the server assigns a thread to each client to process their request commands, allowing them to interact with sessions and other clients.

The client allows users to type in commands using the command line to send messages to the server. This allows them to interact with other clients such as sending messages and creating/joining sessions. The client is implemented with two threads. One to take the client input and execute commands. And the other to listen to the server and process commands given by the server, such as timeouts.
